YouTube Ad Muter & Skipper

A Chrome extension built with vanilla JavaScript — how it works and what I learned building it.

How it works
1
YouTube loads, content.js injects
Chrome injects content.js into every youtube.com tab. The script starts a MutationObserver watching the video player element for class changes.
2
Ad detected via .ad-showing class
YouTube itself adds a CSS class called .ad-showing to the player the moment an ad starts. The observer catches this instantly — no polling needed for detection.
.html5-video-player.ad-showing → ad is playing
3
Video muted immediately
The moment .ad-showing appears, the HTML5 video element is muted. The original mute state is saved first so it can be restored after.
video.muted = true // instant, no delay
4
Skip poller starts (every 100ms)
A setInterval polls every 100ms to try skipping the ad. The skip button only appears after a few seconds, so the code will poll until it's available.
5
Ad skipped by seeking to end
Instead of clicking the skip button (which YouTube blocks), seek the video to duration - 1 second. YouTube sees the ad is almost done and transitions out naturally.
video.currentTime = video.duration - 1
6
Volume restored, poller stopped
When .ad-showing disappears, the interval is cleared and the original mute state is restored. The observer keeps watching for the next ad.
Lessons learned
❌ Didn't work Plain .click() on the skip button
First approach was to find the skip button in the DOM and call .click() on it. YouTube's newer button ignored synthetic clicks entirely — nothing happened.
skipBtn.click() // YouTube ignores this
❌ Still blocked Simulated mouse events
Upgraded to dispatching the full mousedown → mouseup → click event chain to simulate a real human click. YouTube blocked it anyway with "Must be handling a user gesture" — a browser security policy that prevents scripts from faking clicks on certain elements.
skipBtn.dispatchEvent(new MouseEvent('mousedown', { bubbles: true })) skipBtn.dispatchEvent(new MouseEvent('mouseup', { bubbles: true })) skipBtn.dispatchEvent(new MouseEvent('click', { bubbles: true }))
✅ Final solution Seek to video.duration - 1
Switched approach entirely — instead of clicking skip, jump the video playback position to 1 second before the end. YouTube treats it as "ad finished" and exits cleanly. Landing inside already-buffered territory means no network request, no stall.
video.currentTime = video.duration - 1 // smooth, fast, no buffer stall
Key concepts
MutationObserver
A browser API that watches DOM elements for changes. Here it watches the player's class list — zero performance cost since it's event-driven, not polling.
video.duration & currentTime
Native HTML5 video properties. duration is the total ad length in seconds. Setting currentTime scrubs the playback position like dragging a timeline slider.
Video buffer window
Browsers don't download the whole video upfront — just a few seconds ahead of playback. Seeking past the buffer forces a new network request, causing lag.
User gesture policy
Browsers block certain actions (like clicking protected buttons) unless triggered by a real user interaction. Scripts can't fake their way past this — it's a security feature.
YouTube SPA navigation
YouTube is a Single Page App — clicking a new video doesn't reload the page. The extension listens for yt-navigate-finish to re-attach the observer on every navigation.
Manifest V3
Chrome's current extension standard. Content scripts run inside the tab and can touch the DOM. No background service worker needed for this use case.
